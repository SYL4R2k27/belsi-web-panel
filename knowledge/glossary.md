# Glossary — BELSI.Work

## Роли

| Термин | Определение |
|--------|-----------|
| **Installer** (Монтажник) | Полевой работник. Выполняет смены на объектах, загружает фотоотчёты, получает задачи. Работает через мобильное приложение. |
| **Foreman** (Бригадир) | Руководитель бригады. Управляет составом команды, создаёт инвайт-коды для привлечения монтажников. Работает через мобильное приложение. |
| **Curator** (Куратор) | Администратор системы. Контролирует все операции: модерация фото, управление сменами, задачами, финансами, поддержкой. Работает через веб-панель. |

## Бизнес-сущности

| Термин | Определение |
|--------|-----------|
| **Shift** (Смена) | Рабочий период монтажника. Имеет жизненный цикл: active → paused → active → finished/cancelled. Основа для расчёта оплаты. |
| **Shift Photo** (Фото смены) | Фотография, загруженная монтажником во время смены для подтверждения выполнения работ. Привязана к конкретному часу смены (hour_index). |
| **Hour Index** | Порядковый номер часа в рамках смены. Каждая фотография привязана к определённому часу. Уникальная пара (shift_id, hour_index). |
| **Moderation** (Модерация) | Процесс проверки фотоотчётов куратором. Результат: approved (одобрено) или rejected (отклонено с обязательным комментарием). |
| **Task** (Задача) | Рабочий элемент, назначаемый монтажнику куратором. Имеет приоритет, дедлайн и жизненный цикл: new → in_progress → done/cancelled. |
| **Team** (Команда) | Группа монтажников под руководством бригадира. Монтажник может быть в одной активной команде. |
| **Invite Code** (Инвайт-код) | Одноразовый код, создаваемый бригадиром для привлечения монтажника в команду. Имеет срок действия и лимит использований. |
| **Wallet** (Кошелёк) | Виртуальный счёт пользователя для учёта заработка. Один кошелёк на пользователя. |
| **Transaction** (Транзакция) | Движение средств в кошельке: начисление за смену, ручная корректировка, вывод, штраф, бонус. |
| **Payout** (Выплата) | Фактическая выплата средств монтажнику. Содержит детализацию по источникам начисления. |
| **Payout Item** | Отдельная строка в детализации выплаты: за какую смену/бонус/штраф сколько начислено. |
| **Act** (Акт) | Финансово-операционный акт за период. Содержит сумму, статус, привязку к файлу. |
| **Support Ticket** (Тикет поддержки) | Обращение пользователя в поддержку. Имеет категорию, приоритет, статус и историю сообщений. |
| **Internal Note** (Внутренняя заметка) | Сообщение в тикете, видимое только кураторам (is_internal=true). Не видно полевым сотрудникам. |

## Технические термины

| Термин | Определение |
|--------|-----------|
| **Audit Log** (Аудит-лог) | Immutable журнал всех write-действий куратора. Каждая запись: кто, что, когда, старое/новое состояние, IP. Не обновляется и не удаляется. |
| **JWT** (JSON Web Token) | Формат токена авторизации. Access token (короткоживущий) используется для API запросов. Refresh token (долгоживущий) для обновления access token. |
| **RBAC** (Role-Based Access Control) | Управление доступом на основе ролей. Три роли: installer, foreman, curator. Каждая роль имеет определённый набор разрешений. |
| **OTP** (One-Time Password) | Одноразовый пароль для аутентификации мобильных пользователей. Отправляется по SMS. |
| **Signed URL** | Временная ссылка для доступа к файлу в object storage. Содержит подпись и срок действия. Не требует аутентификации для скачивания. |
| **Modular Monolith** | Архитектурный стиль: один deployable процесс, но внутри — изолированные модули с чёткими границами. Промежуточный вариант между монолитом и микросервисами. |
| **Bounded Domain** | Изолированная область ответственности в системе. Каждый домен владеет своими данными и логикой. Пример: shifts, photos, tasks, finance. |
| **Expand/Contract** | Стратегия миграций БД: сначала добавляем новое (expand), затем убираем старое (contract). Обеспечивает backward compatibility. |
| **Optimistic UI** | Паттерн UI: немедленное обновление интерфейса при действии пользователя, до подтверждения сервером. Откат при ошибке. |
| **MSW** (Mock Service Worker) | Инструмент для перехвата HTTP запросов на сетевом уровне в dev-среде. Позволяет разрабатывать frontend без реального backend. |
| **shadcn/ui** | Коллекция UI-компонентов для React. Копируются в проект (не npm-зависимость), полный контроль над кодом. Основа дизайн-системы BELSI.Work. |
| **Partial Index** | Индекс PostgreSQL с условием WHERE. Индексирует только подмножество строк. Пример: индекс активных смен. |
| **PITR** (Point-in-Time Recovery) | Восстановление базы данных на произвольный момент времени. Обеспечивается continuous archiving WAL. |

## Статусы

### Shift Statuses

| Статус | Описание |
|--------|----------|
| `active` | Смена идёт, таймер работает |
| `paused` | Смена на паузе, таймер остановлен |
| `finished` | Смена завершена штатно |
| `cancelled` | Смена отменена |

### Photo Statuses

| Статус | Описание |
|--------|----------|
| `pending` | Фото загружено, ожидает модерации |
| `approved` | Фото одобрено куратором |
| `rejected` | Фото отклонено куратором (с комментарием) |

### Task Statuses

| Статус | Описание |
|--------|----------|
| `new` | Задача создана, не начата |
| `in_progress` | Задача в работе |
| `done` | Задача выполнена |
| `cancelled` | Задача отменена |

### Payment Statuses

| Статус | Описание |
|--------|----------|
| `pending` | Выплата создана, ожидает подтверждения |
| `approved` | Выплата одобрена куратором |
| `paid` | Выплата произведена |
| `cancelled` | Выплата отменена |

### Ticket Statuses

| Статус | Описание |
|--------|----------|
| `open` | Тикет создан, ожидает ответа |
| `in_progress` | Тикет в работе у куратора |
| `resolved` | Тикет решён |
| `closed` | Тикет закрыт |

### Invite Statuses

| Статус | Описание |
|--------|----------|
| `new` | Инвайт создан, не использован |
| `accepted` | Инвайт активирован монтажником |
| `expired` | Срок действия истёк |
| `cancelled` | Инвайт деактивирован вручную |
